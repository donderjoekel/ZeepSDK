Namespace ZeepSDK.Utilities.Override

Classes

ConditionTicker

A singleton MonoBehaviour that manages and ticks IConditionTickable objects according to their specified ConditionTickType. Ticks are performed during Unity's Update, FixedUpdate, LateUpdate, or OnGUI lifecycle methods.

ConditionalOverrideLayer<T>

Represents an override layer that is conditionally active based on a boolean condition. The condition is evaluated periodically according to the specified ConditionTickType, and the override is automatically added or removed from the stack when the condition changes.

OverrideStack<T>

Manages a stack of override layers that can modify a value. Override layers are applied in a stack-based manner, with the most recent active layer taking precedence. Supports both immediate overrides and conditionally active overrides.

Structs

OverrideLayer<T>

Represents a simple override layer that is immediately active when created. Unlike ConditionalOverrideLayer<T>, this override is always active until disposed.

Interfaces

IConditionTickable

Represents an object that can be ticked by the ConditionTicker at regular intervals. Implementations specify when they should be ticked via the TickType property.

IOverrideLayer<T>

Represents a layer in an override stack that can apply a value override. Implementations include OverrideLayer<T> for immediate overrides and ConditionalOverrideLayer<T> for conditionally active overrides.

Enums

ConditionTickType

Specifies when a condition should be evaluated in the Unity lifecycle. Used by ConditionalOverrideLayer<T> to determine when to check its condition.
